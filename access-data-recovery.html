List of common Error Messages

MS Access database is also prone to corruption. When the Access database is corrupted, it can show different types of error messages when the user tries to open the DB files.You can use this software to resolve these errors.Some of the most common error messages are listed here:

  1. Error 3343- Unrecognized Database Format 'filename.mdb'.
  2. Error 2239- The database 'filename.mdb' needs to be repaired or isn't a database file.
  3. Error 3312- Record(s) cannot be read; no read permission on 'xxxx'.
  4. Error 3049- Cannot open database 'filename.mdb'. It may not be a database that your application recognizes, or the file may be corrupt.
  5. Error 3011- The Microsoft Jet database engine could not find the object 'xxxx'. Make sure the object exists and that you spell its name and the path name correctly.
  6. Error 3033- You do not have the necessary permissions to use the 'filename.mdb' object. Have your system administrator or the person who created the object establish the appropriate permissions for you.
  7. Error 35012- Unexpected error.
  8. Error 3259- Invalid field data type.
  9. Error 3022-The changes you requested to the table were not successful because they would create duplicate values in the index, primary key, or relationship.
  10. Error 9505-Microsoft Access has detected that this database is in an inconsistent state, and will attempt to recover the database. During this process, a backup copy of the database will be made and all recovered objects will be placed in a new database. Access will then open the new database. The names of objects that were not successfully recovered will be logged in the “Recovery Errors” table.
  11. Microsoft Access asks for a password when opening a corrupt database.
  12. Error 29081-The database cannot be opened because the VBA project contained in it cannot be read. The database can be opened only if the VBA project is first deleted. Deleting the VBA project removes all code from modules, forms, and reports. You should back up your database before attempting to open the database and delete the VBA project.
